## Overview

This repository contains my Week 1 task for the Network Administration Internship Program. The goal of this week was to build a strong foundation in basic networking concepts and gain hands-on experience with two industry standard network simulation tools, Cisco Packet Tracer and GNS3.

A simple LAN topology was built in both tools. Devices were assigned static IP addresses. The ping command was used to test and confirm connectivity between them.

## What This Repo Contains

- `Report.pdf` - Full report covering networking concepts, interface overviews, ping test results, challenges faced, and conclusion
- `Cisco Packet Tracer.pkt` - Packet Tracer project file with the LAN topology
- `GNS3 Project/` - GNS3 project files with the LAN topology
- `Screenshots/` - Screenshots of both topologies and ping test results

## Networking Concepts Covered

- Networks and LAN (Local Area Network)
- IP addressing and static IP addressing
- Network devices: router, switch, PC/end device
- Communication models (OSI and TCP/IP)
- Ping command for connectivity testing

## Tools Used

- Cisco Packet Tracer
- GNS3 (Graphical Network Simulator-3)
- PuTTY

## Topology Summary

A basic LAN was built with a router, a switch, and multiple PCs/end devices in both Packet Tracer and GNS3. Static IP addresses were assigned to each device, and ping tests were run between all devices to confirm connectivity. All ping tests were successful with 0% packet loss.

## Key Learning

I was already familiar with Cisco Packet Tracer from previous coursework, so building and configuring the topology there was straightforward. GNS3 was new to me. I learned the interface, added and connected switches and client PCs, configured IP addressing through the CLI, and verified connectivity using ping.
